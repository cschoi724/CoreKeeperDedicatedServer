# Windows 운영 자동화 설계

이 문서는 Windows 노트북에서 Core Keeper Dedicated Server를 설치하고 실행하는 운영 자동화 방향을 기록한다. 현재는 설계 문서이며 실제 스크립트 구현은 아직 하지 않았다.

## 확정된 기본값

- 서버 설치 경로: `C:\CoreKeeperServer`
- 백업 경로: `D:\Backups\CoreKeeper`
- SteamCMD 설치 방식: 템플릿이 자동 다운로드/설치
- 서버 접속 방식: Steam 전용 SDR(Game ID)
- 서버 실행 계정: 현재 Windows 로그인 사용자
- 절전모드 설정: 자동 변경하지 않고 문서로 안내

## 실행 방식

기본 실행 방식은 수동 실행이다.

사용자는 설치 후 필요할 때 서버 시작 스크립트를 직접 실행한다. 서버가 정상 실행되면 Game ID를 확인하고 친구들에게 공유한다.

## 자동 실행 옵션

자동 실행은 기본값이 아니라 사용자가 켜고 끌 수 있는 옵션으로 제공한다.

구현 후보:

- `register-task.ps1`: Windows Task Scheduler 작업 등록
- `unregister-task.ps1`: 등록된 자동 실행 작업 제거
- `enable-task.ps1`: 등록된 작업 활성화
- `disable-task.ps1`: 등록된 작업 비활성화

작업 이름 후보:

```text
CoreKeeperServer
```

## 특정 시간 서버 재시작

서버 안정 운영을 위해 특정 시간에 서버를 재시작하는 옵션을 제공한다.

추천 설계:

- 기본값은 재시작 예약 없음
- 사용자가 시간을 입력하면 Task Scheduler에 별도 작업을 등록
- 재시작 작업 이름 후보: `CoreKeeperServerRestart`
- 시간 형식 후보: `HH:mm`, 예: `05:00`

재시작 흐름:

1. 서버가 실행 중인지 확인한다.
2. 가능하면 정상 종료 명령을 사용한다.
3. 정상 종료 방식이 검증되지 않았으면 사용자에게 수동 종료 안내를 우선한다.
4. 서버 시작 스크립트를 다시 실행한다.

주의:

- Core Keeper Dedicated Server의 안전한 종료 방식은 Windows 실기 검증에서 확인해야 한다.
- 검증 전에는 강제 종료 기반 자동 재시작을 기본값으로 넣지 않는다.

## 절전모드 안내

서버를 24시간 열어둘 경우 Windows 절전모드와 최대 절전모드는 꺼야 한다.

현재 결정:

- 자동화 스크립트가 전원 설정을 임의로 변경하지 않는다.
- README 또는 운영 문서에서 사용자가 직접 확인하도록 안내한다.

## 관리자 권한

일반 실행은 현재 Windows 사용자 권한으로 진행한다.

관리자 권한이 필요할 수 있는 작업:

- Task Scheduler에 부팅 시 실행 작업 등록
- 일부 경로 권한 변경
- 향후 Direct Connect 방화벽 규칙 추가

현재 범위에서는 Direct Connect를 제외하므로 방화벽 자동 변경은 구현하지 않는다.
