# Additional Research & Design Review

## 목적

Core Keeper Dedicated Server 구축 자동화 프로젝트 진행 중이다.

현재 서버 설치, 월드 이전, 자동 시작 등의 기능 외에도 실제 장기간 운영 시 필요한 동작 방식을 정확히 이해하고 싶다.

구현에 들어가기 전에 아래 항목들을 조사하고 문서화해줘.

---

# 1. 플레이어가 없을 때 월드 동작 방식 조사

현재 가장 궁금한 부분이다.

Dedicated Server 실행 중

```text
플레이어 0명
↓
서버는 실행 상태
```

일 때 월드가 어떻게 동작하는지 확인해줘.

## 조사 항목

* 월드 시간이 계속 진행되는지
* 월드가 Pause 상태가 되는지
* 농작물 성장 여부
* 자동 채굴 드릴 동작 여부
* 컨베이어 동작 여부
* 용광로 동작 여부
* 자동화 설비 동작 여부
* 몬스터 스폰 여부
* 몬스터 AI 진행 여부
* 일반 플레이어 호스트 방식과 Dedicated Server 방식의 차이

## 결과 정리

다음 질문에 답할 수 있게 정리해줘.

```text
아무도 접속하지 않을 때
월드는 정지하는가?
계속 진행되는가?
```

만약 설정 옵션이 존재한다면 설정 방법도 정리해줘.

---

# 2. 저장(Save) 시스템 조사

일반 플레이에서는

> 저장 후 종료

기능이 존재한다.

Dedicated Server에서는 저장이 어떤 방식으로 처리되는지 조사해줘.

## 조사 항목

* 자동 저장 존재 여부
* 저장 주기 존재 여부
* 수동 저장 기능 존재 여부
* 서버 종료 시 저장 방식
* 비정상 종료 시 저장 손실 가능성
* 업데이트 중 저장 처리 방식
* 권장 종료 절차

---

# 3. 캐릭터 데이터와 월드 데이터 구조 조사

현재 확인된 상황

```text
친구가 자기 개인 월드에서 키운 캐릭터를
내 월드로 그대로 가져와 플레이 가능
```

이를 기준으로 저장 구조를 분석해줘.

## 캐릭터

### 조사 항목

* 저장 위치
* 저장 방식
* Steam Cloud 사용 여부
* 로컬 저장 여부
* 인벤토리 저장 위치
* 장비 저장 위치
* 스킬 저장 위치
* 사망 정보 저장 위치

## 월드

### 조사 항목

* 저장 위치
* 저장 방식
* Dedicated Server 이전 가능 여부
* 월드 파일 구조

## 관계

캐릭터와 월드가

```text
완전히 분리
```

되어 있는지 확인해줘.

# 4. 서버 종료 정책 검토

현재 계획

## 일반 종료

```text
서버 종료
↓
저장
↓
정상 종료
```

## 자동 점검

```text
저장
↓
종료
↓
업데이트
↓
재시작
```

## 자동 재시작

```text
비정상 종료일 때만 재시작
```

위 정책이 적절한지 검토해줘.

---

# 5. Graceful Shutdown 검토

게임 서버 운영 관점에서 아래 기능이 필요한지 검토해줘.

## 목표

### 서버 종료

```text
서버 종료
↓
저장
↓
정상 종료
```

### 자동 점검

```text
저장
↓
종료
↓
업데이트
↓
재시작
```

### 자동 재시작

```text
저장 가능한 경우 저장
↓
재시작
```

## 확인 항목

* 월드 손상 방지 효과
* 저장 데이터 유실 방지 효과
* Core Keeper Dedicated Server에서 실제 필요한 기능인지
* 구현 방법

# 6. 운영 기능 추가 검토

현재 우선 고려 중인 기능

## 서버 시작 정보 출력

출력 항목

```text
World Name
World ID
Game ID
Save Path
Server Start Time
Max Players
```

특히

```text
Game ID
```

변경 여부를 쉽게 확인할 수 있어야 한다.

---

## Watchdog 자동 재시작

옵션 기능

기본값

```text
비활성화
```

활성화 시

```text
비정상 종료
↓
자동 재시작
```

정상 종료는 재시작하지 않는다.

---

## 자동 백업

예시

```text
일일 백업
업데이트 전 백업
수동 백업
```

백업 보관 정책도 검토해줘.

* 충돌 감지
* 백업 완료
* 업데이트 시작
* 업데이트 완료

Game ID

# 7. 자동 점검 및 자동 업데이트 검토

예시

```text
매일 지정 시간
↓
서버 종료
↓
업데이트 확인
↓
업데이트 적용
↓
서버 재시작
```

## 확인 필요

* Core Keeper 업데이트가 기존 월드에 정상 반영되는지
* 월드 손상 위험은 없는지
* 실제 운영 가치가 있는지

필요성이 낮다면 구현 대신 문서화만 해도 된다.
