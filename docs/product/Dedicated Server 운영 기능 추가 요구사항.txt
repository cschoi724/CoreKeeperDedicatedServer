# Dedicated Server 운영 기능 추가 요구사항

## 개요

현재 Core Keeper Dedicated Server 구축 자동화 프로젝트를 진행 중이다.

기존 목표(설치, 월드 이전, 자동 시작, 문서화)는 유지하고, 실제 운영 편의성을 위한 기능을 추가하고 싶다.

구현 방법은 자유롭게 설계하되 아래 목적을 만족하도록 구성해줘.

---

# 목표

단순히 서버를 설치하는 수준이 아니라,

**개인 또는 소규모 그룹이 장기간 안정적으로 운영할 수 있는 서버 관리 도구**

를 만드는 것이 목적이다.

운영자는 서버 콘솔을 항상 보고 있지 않기 때문에

* 상태 확인
* 장애 복구
* 백업
* 알림

기능을 중요하게 생각한다.

---

# 우선 구현 희망 기능

## 1. 서버 시작 정보 출력

서버 시작 시 아래 정보가 콘솔 및 로그에 출력되도록 구성

```text
World Name
World ID
Current Game ID
Save Path
Server Start Time
Max Players
```

특히

```text
Game ID
```

가 변경되었는지 쉽게 확인할 수 있어야 한다.

---

## 2. Watchdog 기반 자동 재시작

설정 옵션으로 제공

### 기본값

```text
자동 재시작 비활성화
```

### 활성화 시

```text
서버 프로세스 비정상 종료
↓
자동 재시작
```

### 요구사항

* 서버 프로세스가 비정상 종료되면 자동 재시작
* 정상 종료 명령으로 종료한 경우에는 재시작하지 않음
* 재시작 횟수 및 시간을 로그에 기록

예시

```text
Server crashed
Restarting in 30 seconds...
Restart count: 3
```

---

## 3. 자동 백업

운영 안정성 측면에서 중요도가 높음

### 지원 기능

* 지정 시간 자동 백업
* 서버 업데이트 전 자동 백업
* 수동 백업

### 백업 정책 예시

```text
일일 백업
최근 30개 보관
오래된 백업 자동 삭제
```

### 추가 검토

* 백업 검증 기능
* 백업 복원 기능
* 최신 백업 확인 기능

---

## 4. Discord Webhook 알림

Webhook URL 설정 시 활성화

### 알림 이벤트

* 서버 시작
* 서버 종료
* 서버 재시작
* 서버 충돌 감지
* 백업 완료
* 업데이트 시작
* 업데이트 완료

### 서버 시작 시 포함 정보

```text
Game ID
World Name
Start Time
```

### 예시

```text
🟢 Core Keeper Server Started

Game ID: XXXXX
World: MyWorld
Time: 2026-06-24 20:00
```

---

## 5. 상태 조회 기능

운영자가 빠르게 상태 확인 가능해야 함

### 예시

```text
status
```

### 출력

```text
Server Status : Running
Uptime : 3d 12h
Game ID : XXXXX
Restart Count : 1
Last Backup : 2026-06-24 05:00
```

# 자동 점검 및 자동 업데이트 검토

## 목표

```text
지정 시간
↓
서버 저장
↓
정상 종료
↓
업데이트 확인
↓
업데이트 적용
↓
서버 재시작
```

## 반드시 확인할 내용

* Core Keeper 업데이트가 기존 월드에 정상 반영되는지
* 자동 업데이트 시 월드 손상 위험이 없는지
* 실제 운영 가치가 있는지
* Dedicated Server 운영 시 일반적으로 사용되는 방식인지

### 판단 기준

가치가 낮으면

```text
문서화만 수행
```

해도 됨.

* Discord 알림
* Watchdog 자동 재시작
* 자동 백업

---

## Priority 2

권장 구현 후보

* 상태 조회 기능

* 백업 복원 기능

---

## Priority 3

검토 후 구현

* 자동 점검
* 자동 업데이트
* 서버 상태 페이지

---

# 설계 검토 요청

일반적인 게임 전용 서버 운영 관점에서

다음 조건을 만족하는 추가 기능이 있다면 제안해줘.

## 조건

* 개인 서버 운영에 실제 도움이 되는 기능
* 유지보수 부담이 크지 않은 기능
* Core Keeper 운영에 적합한 기능

## 제외

* 과도하게 복잡한 기능
* 엔터프라이즈 수준 기능
* 실사용 가치가 낮은 기능

---

# 중요

구현 전에

* Core Keeper Dedicated Server 특성
* 저장 방식
* 월드 구조
* 업데이트 방식
* 일반 게임 서버 운영 사례

를 조사한 뒤 설계해줘.

추측이 아닌 실제 확인된 정보와 일반적인 운영 관행을 구분해서 작성해줘.
