# automation 결정사항

## 확정된 결정

### 2026-06-24: 자동화 대상은 Windows 노트북

- 결정: 자동화 템플릿은 Windows에서 clone 후 실행하는 것을 목표로 한다.
- 근거: 사용자 요청에서 실제 서버 실행 환경이 집의 Windows 노트북이라고 명시됨.

### 2026-06-24: 현재 세션에서는 구현하지 않음

- 결정: 이 세션에서는 PowerShell/Batch 스크립트를 만들지 않는다.
- 근거: 사용자 요청에서 루트 관리 에이전트이며 실제 구현은 하지 말라고 명시됨.

### 2026-06-24: Steam 전용 접속만 지원

- 결정: 현재 자동화 템플릿은 Steam 접속자용 SDR(Game ID) 흐름을 우선 지원한다.
- 근거: 사용자가 Steam에서만 접속한다고 명시함.
- 영향: Direct Connect, 포트포워딩, Windows 방화벽 자동 생성은 현재 구현 범위에서 제외한다.

### 2026-06-24: 기본 빈 월드와 단일 파일 import

- 결정: 서버 노트북에는 기존 월드가 없으므로, 월드 자동화는 기본 빈 월드를 우선하고 사용자가 가져온 단일 `.world.gzip` 파일을 import하는 흐름으로 설계한다.
- 근거: 사용자가 Windows 노트북에서 Core Keeper를 실행한 적이 없고 기존 월드도 없다고 설명함.
- 세부 설계: `WORLD_MIGRATION_DESIGN.md`

### 2026-06-24: 설치/백업/실행 운영 기본값

- 결정: 서버 설치 경로는 `C:\CoreKeeperServer`, 백업 경로는 `D:\Backups\CoreKeeper`, SteamCMD는 템플릿에서 자동 설치한다.
- 근거: 사용자가 추천 경로와 추천 설치 방식을 승인함.

### 2026-06-24: 기본 수동 실행과 선택 자동 실행/재시작

- 결정: 서버는 기본 수동 실행이며, 자동 실행과 특정 시간 재시작은 사용자가 켜고 끌 수 있게 설계한다.
- 근거: 사용자가 기본은 수동 실행, 자동 실행 온/오프, 특정 시간 서버 재시작을 원한다고 설명함.
- 세부 설계: `OPERATIONS_DESIGN.md`

### 2026-06-24: 별도 Windows 계정과 전원 설정 자동 변경 제외

- 결정: 별도 Windows 계정은 만들지 않고 현재 Windows 로그인 사용자로 실행한다. 절전모드는 자동 변경하지 않고 문서로 안내한다.
- 근거: 사용자가 별도 계정을 만들지 않고 절전모드는 문서 안내만 원한다고 답함.

## 후보 결정

- PowerShell을 기본 스크립트 언어로 사용
- SteamCMD 기반 설치/업데이트 사용
- SDR(Game ID)를 기본 접속 방식으로 사용
- Task Scheduler로 부팅 자동 시작 등록
