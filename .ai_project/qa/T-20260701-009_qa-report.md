## 확인 결과

- `automation/src/Games/Valheim/game.json`가 추가됐고 `gameId`는 `valheim`이다.
- `automation/config/games/valheim.example.json`가 추가됐다.
- Core Keeper 외 두 번째 manifest가 `automation/src/Games/**/game.json` 구조에서 정적으로 발견된다.
- Valheim manifest는 필수 top-level 필드 `gameId`, `displayName`, `steam`, `paths`, `server`, `backup`, `features`를 포함한다.
- 미구현 기능은 `features.worldImport`, `features.configPatch`, `features.gracefulStop`, `features.healthCheck` 모두 `false`로 명시됐다.
- `ADAPTER_GUIDE.md`는 Valheim을 실제 서버 지원 완료가 아니라 manifest discovery 검증용 skeleton으로 설명한다.
- 현재 환경에 `pwsh`/`powershell`이 없어 PowerShell 원 검증 명령은 실행하지 못했다.

```bash
ruby -rjson -e 'ARGV.each { |p| JSON.parse(File.read(p)); puts "#{p}: JSON OK" }' automation/src/Games/CoreKeeper/game.json automation/src/Games/Valheim/game.json automation/config/games/corekeeper.example.json automation/config/games/valheim.example.json
ruby -rjson -e 'required=%w[gameId displayName steam paths server backup features]; Dir["automation/src/Games/**/game.json"].sort.each do |p| j=JSON.parse(File.read(p)); missing=required.reject{|k| j.key?(k)}; raise "#{p} missing #{missing.join(",")}" unless missing.empty?; raise "#{p} missing steam.appId" unless j.dig("steam","appId").to_s.size>0; raise "#{p} missing steam.login" unless j.dig("steam","login").to_s.size>0; raise "#{p} missing launchCandidates" unless j.dig("server","launchCandidates").is_a?(Array) && j.dig("server","launchCandidates").any?; raise "#{p} missing backup.targets" unless j.dig("backup","targets").is_a?(Array) && j.dig("backup","targets").any?; puts "#{j["gameId"]}: manifest contract OK"; end'
```

## 잔여 리스크

- Valheim Dedicated Server AppID `896660`은 skeleton 후보로 기록됐지만, Dev Report의 Steam Store appdetails API 결과는 `success:false`였으므로 AppID 유효성은 Windows/SteamCMD 환경에서 `app_update 896660 validate`로 재확인해야 한다.
- 기본 포트와 실행/데이터 경로 후보는 문서에서 Windows/SteamCMD 검증 필요 항목으로 남아 있다. 실제 Valheim 운영 지원 전에는 후보값을 확정하지 않는다.
- 현재 작업 트리에는 이전 GSM-R 작업의 광범위한 변경이 함께 남아 있어, 이번 QA는 Valheim skeleton 산출물과 관련 문서 범위만 판정했다.

## 결론

Core가 Core Keeper에만 종속되지 않는지 확인하기 위한 두 번째 Adapter skeleton 목표는 충족한다. 실제 Valheim 서버 설치/실행 지원은 범위 밖이므로 Windows/SteamCMD 검증 리스크를 남기고 `qa_passed` 처리 가능하다.
