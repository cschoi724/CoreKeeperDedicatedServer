# automation 결정사항

## 확정된 결정

### 2026-06-24: 자동화 대상은 Windows 노트북

- 결정: 자동화 템플릿은 Windows에서 clone 후 실행하는 것을 목표로 한다.
- 근거: 사용자 요청에서 실제 서버 실행 환경이 집의 Windows 노트북이라고 명시됨.

### 2026-06-24: 현재 세션에서는 구현하지 않음

- 결정: 이 세션에서는 PowerShell/Batch 스크립트를 만들지 않는다.
- 근거: 사용자 요청에서 루트 관리 에이전트이며 실제 구현은 하지 말라고 명시됨.

### 2026-06-24: Steam 전용 접속만 지원

- 결정: 현재 자동화 템플릿은 Steam 접속자용 SDR(Game ID) 흐름을 우선 지원한다.
- 근거: 사용자가 Steam에서만 접속한다고 명시함.
- 영향: Direct Connect, 포트포워딩, Windows 방화벽 자동 생성은 현재 구현 범위에서 제외한다.

### 2026-06-24: 기존 월드는 사용자 Steam 계정에서 이전

- 결정: 월드 이전 자동화는 사용자 Steam 계정 저장 폴더에서 월드를 선택해 Dedicated Server 저장 폴더로 복사하는 흐름으로 설계한다.
- 근거: 사용자가 기존 월드가 본인 Steam 계정에 속해 있다고 설명함.
- 세부 설계: `WORLD_MIGRATION_DESIGN.md`

## 후보 결정

- PowerShell을 기본 스크립트 언어로 사용
- SteamCMD 기반 설치/업데이트 사용
- SDR(Game ID)를 기본 접속 방식으로 사용
- Task Scheduler로 부팅 자동 시작 등록
